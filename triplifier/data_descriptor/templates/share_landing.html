On this page you will find various options to share metadata about your data in a privacy-preserving manner.


Download your local semantic mapping

Given you have previously uploaded a Flyover semantic map JSON-LD file, you can now download your local semantic mapping.

You can use this file to archive your mapping work and to share it with others.
We strongly recommend archiving your mapping properly, as it can be easily be re-purposed in case your data and or mapping needs to be updated.



What is a 'local semantic map'?
A local semantic map defines how your data's terminology maps to the global semantic map, which defines standardised variable definitions, data types, and value mappings that can be consistently applied across different datasets to ensure semantic interoperability.

Download your local ontology file

Ran into variables you could not directly map to the global mapping, or have not used a global map at all?
In that case we recommend that you download your local ontology.



What is an 'ontology file'?
The ontology file describes your data structure, meaning it shows what your column names exactly are, even if you did not describe or map them.
This can be incredibly useful if you have added data that does not directly correspond to any concepts in a global semantic map.
The ontology file only contains variable names, value names and counts, and the file does not contain any row-level data.
